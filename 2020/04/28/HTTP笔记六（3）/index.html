HTTP笔记六（3）

Share

请求首部字段

  1. Accept
    作用：通知服务器，用户代理能够处理的媒体类型及媒体类型的相对优先级，使用type/subtype指定。
    若想要给显示的媒体类型增加优先级，则使用 q= 来额外表示权重值 ，用分号进行分隔。权重值 q 的范围是 0~1（可精确到小数点后3位），且1为最大值。不指定权重 q 值时，默认权重为 q=1.0。
  2. Accept-Charset
    作用：通知服务器用户代理支持的字符集及字符集的相对优先顺序，同样用 q 表示权重。
  3. Accept-Encoding
    作用：告知服务器用户代理支持的内容编码及内容编码的优先级顺序，同样用 q 表示权重。
  4. Accept-Language
    作用：告知服务器用户代理能够处理的自然语言集（指中文或英文等），以及相对优先级，同样用 q 表示权重。
  5. Authorization
    作用：告知服务器，用户代理的认证信息（证书值）。
  6. Expect
    作用：告知服务器期望出现的某种特定行为，若无法做出回应发生错误，返回状态码 417 Expectation Failed。
  7. From
    作用：告知服务器使用用户代理的用户的电子邮件地址。
  8. Host
    作用：告知服务器请求的资源所处的互联网主机名和端口号，必须包含在请求内，用来区分同一个IP上的不同主机。
  9. If-Match
    作用：告知服务器匹配资源所用的实体标记（ETag）值。这时的服务器无法使用弱 ETag 值，与资源的 ETag 值一致时执行请求。使用*会忽略 ETag 值，只要资源存在就处理请求。
  10. If-Modified-Since
    作用：告知服务器若 If-Modified-Since 字段值早于资源的更新时间，则希望能处理该请求。
    若在指定日期之后都没更新，，则返回状态码 304 Not Modified 的响应。
  11. If-None-Match
    作用：与If-Match作用相反，指定 If-None-Match 字段值的实体标记（ETag）值与请求资源的 ETag 不一致时，它就告知服务器处理该请求。
  12. If-Range
    作用：告知服务器若指定的 IfRange 字段值（ETag 值或者时间）和请求资源的 ETag 值或时间相一 致时，则作为范围请求处理。反之，则返回全体资源。（更新资源时若不使用If-Range则需要处理两次）
  13. If-Unmodified-Since
    作用： 和首部字段 If-Modified-Since 的作用相反。告知服务器，指定的请求资源只有在字段值内指定的日期时间之后，未发生更新的情况下，才能处理请求。如果在指定日期时间后发生了更新，则以状态码 412 Precondition Failed 作为响应返回。
  14. Max-Forwards
    作用：用来掌握服务器传输路径的通信状态，每经过一个服务器后面的值就-1。
  15. Proxy-Authorization
    作用：接收到从代理服务器发来的认证质询时，客户端告知服务器认证所需要的信息。
  16. Range
    作用：告知服务器资源的指定范围。处理之后返回206 Partial Content，无法处理返回200 OK。
  17. Referer
    作用：告知服务器请求的原始资源的URI。
  18. TE
    作用：告知服务器客户端能够处理响应的传输编码方式及相对优先级。
  19. User-Agent
    作用：创建请求的浏览器和用户代理名称等信息传达给服务器。
