HTTP/1.1通用首部字段

  1. Cache-Control
    作用：能够控制缓存的行为，可多指令，用逗号隔开。
    缓存请求指令：

缓存响应指令：

拓展：cache-extension token
通过 cache-extension 标记（token），可以扩展 Cache-Control 首部字段内的指令。
2. Connection
作用：控制不再转发给代理的首部字段（通过后接相应的字段名，代理服务器会删除他之后再转发）；
管理持久连接（Close断开连接，Keep-Alive持续连接）。
3. Date
作用：表明创建 HTTP 报文的日期和时间。
4. Pragma
作用： HTTP/1.1 之前版本的历史遗留字段，仅作为与 HTTP/1.0 的向后兼容而定义。
通常包含以下两个首部字段，用在客户端发送的请求中，要求中间服务器不返回缓存的资源。

  1. Trailer
    作用：事先说明在报文主体后记录了哪些首部字段。该首部字段可应用在 HTTP/1.1 版本分块传输编码时。
  2. Transfer-Encoding
    作用：规定了传输报文主体时采用的编码方式。 （HTTP/1.1 的传输编码方式仅对分块传输编码有效。）
  3. Upgrade
    作用：检测 HTTP 协议及其他协议是否可使用更高的版本进行通信，其参数值可以用来指定一个完全不同的通信协议。（服务器用101 Switching Protocols状态码作为响应返回）
  4. Via
    作用：追踪客户端与服务器之间的请求和响应报文的传输路径。（经常会和TRACE方法一起使用）
  5. Warning
    作用：告知用户一些与缓存相关的问题的警告。
    Warning: [警告码][警告的主机:端口号]“[警告内容]”([日期时间])
