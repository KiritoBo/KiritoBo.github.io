组件数据的存放

组件对象也有自己的data属性，且data属性必须是一个函数，而且这个函数返回一个对象，对象内部保存这数据。

如果data不是函数，多次使用一个组件他们之间就会互相影响。

父子组件通信

  1. 父组件向子组件传递数据（通过props向子组件传递数据）

    • props可以以字符串数组形式列出prop：
      props: ['title', 'likes', 'isPublished', 'commentIds', 'author']
    • props可以以对象形式列出prop，这些属性的名称和值分别是prop各自的名称和类型：
    • prop 通过v-bind动态赋值
      <cpn v-bind:title="post.title"></cpn>
    • prop验证
      为了定制 prop 的验证方式，你可以为 props 中的值提供一个带有验证需求的对象，而不是一个字符串数组。
    • HTML 中的属性名是大小写不敏感的，所以浏览器会把所有大写字符解释为小写字符。这意味着当你使用 DOM 中的模板时，驼峰命名法的 prop 名需要使用其等价的 kebab-case (短横线分隔命名) 命名。

  2. 通过自定义事件($emit Events)向父组件发送消息
    子组件通过this.$emit('my-event')发送事件；父组件通过v-on监听事件。（事件名使用kebab-case命名，不用驼峰）

父子组件的访问方式

  • 父组件访问子组件：$children或$refs
    $children[n]通过n的值获取第n+1个组件，是数组类型，此方法不常用。
    $refs通过下面的方式获取组件，是对象类型。
    <cpn ref="aaa"></cpn>
    this.$refs.aaa.name
  • 子组件访问父组件：$parent（不常用） $root（直接访问Vue实例）
