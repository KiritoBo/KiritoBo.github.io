事件监听v-on

  • 缩写：@
  • 预期：Function | Inline Statement | Object
  • 参数：event

用在普通元素上时，只能监听原生 DOM 事件。用在自定义元素组件上时，也可以监听子组件触发的自定义事件。

在监听原生 DOM 事件时，需要注意参数问题：

  1. 如果该方法不需要额外参数那么方法后面的()可以不加 即btnClick后面可不写()
    <button @click="btnClick">按钮</button>
    注：如果方法本身中有一个参数，那么会默认将原生事件event参数传递进去。
  2. 如果需要传入某个参数同时需要event时，可以通过$event（手动获取event）传入事件。
    <button @click="btnClick('abc',$event)">按钮</button>

v-on修饰符

  • .stop - 调用 event.stopPropagation()。 @click.stop阻止冒泡
  • .prevent - 调用 event.preventDefault()。 阻止默认事件
  • .{keyCode | keyAlias} - 只当事件是从特定键盘按键触发时才触发回调。
  • .native - 监听组件根元素的原生事件。
  • .once - 只触发一次回调。
  • .capture - 添加事件侦听器时使用 capture 模式。
  • .self - 只当事件是从侦听器绑定的元素本身触发时才触发回调。
  • .left - (2.2.0) 只当点击鼠标左键时触发。
  • .right - (2.2.0) 只当点击鼠标右键时触发。
  • .middle - (2.2.0) 只当点击鼠标中键时触发。
  • .passive - (2.3.0) 以 { passive: true } 模式添加侦听器。

条件判断 v-if v-else-if v-else

v-if的值为true时渲染元素，否则执行后面的v-else或v-else-if。
v-else前一兄弟必须有v-if或v-else-if，后面不需要条件。
v-else-if前一兄弟必须有v-if或v-else-if。

登录切换小案例

v-show

与v-if类似，用于决定一个元素是否渲染。
v-if当条件为flase时，压根不会有对应的元素在dom中。（只有一次切换用）
v-show当条件为flase时，仅仅是将元素的display属性设置为none。（频繁显示隐藏使用）

v-for 循环遍历

预期：Array | Object | number | string | Iterable (2.6 新增)
渲染一组数据时使用，语法类似js中的for循环。
格式：item in items

也可以获取数组索引值

在遍历对象过程中，如果只要获取一个值，得到的是value

获取key和value 格式：（value,key）

获取key value index 格式：(value,key,index)

v-for 的默认行为会尝试原地修改元素而不是移动它们。要强制其重新排序元素，你需要用特殊 attribute key 来提供一个排序提示:(加了key之后性能会提高)

当和 v-if 一起使用时，v-for 的优先级比 v-if 更高。

数组中那些方法是响应式的

  1. push 在最后加一个元素（可一次添加多个元素）
  2. pop 删除数组最后一个元素
  3. shift 删除数组第一个元素
  4. unshift 在数组最前面加一个元素（可一次添加多个元素）
  5. splice 作用：删除 插入 替换元素
    第一个参数表示从哪个位置开始
    删除元素：第二个参数传入你要删除几个元素（如果没有传，就删除后面所有元素）
    替换元素：第二个参数表示要替换几个元素，第三个参数 要替换成的元素
    插入元素：第二个参数是0 第三个参数是要插入的元素
  6. reverse 反转元素

注意：通过索引修改数组不是响应式,如下面这种：
this.letters[0]='bbbb';
解决办法：vue.set(要修改的对象，索引值，修改后的值）也是响应式
